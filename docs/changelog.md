# 更新日志

## 2026-04-29（续）

### 新增功能

- **布局分列开关**：在"主题样式 → 布局分列"分类新增启用/禁用开关
  - 开关位于展示列数按钮左侧，关闭时列数/行数按钮变灰不可点击
  - 关闭后恢复原版 TAB 布局，不再分页
  - 配置持久化到 `neotab-layout.json`

### UI 改进

- **配置界面 Tooltip 换行**：所有悬浮提示文字超出宽度时自动换行（最大宽度 200px）
- **翻页图标优化**：将分页翻页箭头从像素风实心三角形改为高分辨率 Chevron（`<` `>`）图标
  - 使用 80×128px 纹理缩放至 10×16px 渲染，GPU 双线性过滤，视觉平滑
  - 图标颜色根据 TAB 背景色亮度自适应：亮色背景显示深色图标，暗色背景显示浅色图标
  - 带 1px 黑色阴影偏移，增强任意背景下的可读性

## 2026-04-29

### 问题修复

- **修复配置界面在高GUI缩放下超出屏幕边界的问题**
  - 问题：GUI缩放设置为6时，配置界面右边框和HSV颜色选择器超出屏幕右边界
  - 修复：实现响应式布局，根据屏幕宽度动态调整内容区域宽度和控件位置
  - 效果：在所有GUI缩放级别（自动、1-6）下都能正常显示和操作
  - 技术改进：
    - 使用动态边距计算（最小16px），为内容区域留出更多空间
    - 预先计算颜色选择器宽度，确保不超出右边界
    - 设置内容宽度范围（200px-360px），适应不同屏幕尺寸
    - 添加边界检查，防止控件超出可视区域

### 新增功能

- **动态布局限制系统**：TAB 列表的最大列数和行数现在根据 GUI 缩放和血量显示模式动态调整
  - 实现固定映射表：为每个 GUI 缩放级别（自动、1-6）和血量显示模式（单独/完整）预设安全上限
  - 创建 `TabLayoutLimits` 类：提供动态限制查询和推荐值计算
  - 创建 `LayoutConfigMonitor` 类：采用事件驱动方式监听 GUI 缩放和血量模式变化
  - 性能优化：使用缓存机制，只在变化时调整配置（每秒最多 20 次检查，正常情况零开销）

- **布局分列配置**：在"主题样式"模块新增"布局分列"section
  - 展示列数按钮：循环调整列数（1 → 2 → 3 → ... → 最大值 → 1）
  - 展示行数按钮：循环调整行数（5 → 10 → 15 → 20 → 25 → 30 → 35 → 40 → 5，不超过当前最大值）
  - 按钮显示当前值（例如"展示列数: 3"）
  - 最大值根据当前 GUI 缩放和血量显示模式自动调整

### 问题修复

- **修复 GUI 缩放变化时 TAB 列表显示未自动调整的问题**
  - 问题：用户更改 GUI 缩放后（例如从 5 变为 6），TAB 列表仍使用旧的行列配置，导致内容挤压
  - 修复：通过 `LayoutConfigMonitor` 监听 GUI 缩放变化，自动调整配置到新的限制范围内
  - 效果：GUI 缩放变化后，最多 1 个 tick（50ms）内自动调整，无需用户手动操作

- **修复血量显示模式变化时 TAB 列表显示未自动调整的问题**
  - 问题：切换血量显示模式（单独↔完整）时，最大列数变化，但 TAB 列表仍使用旧配置
  - 修复：监听血量模式变化，自动调整列数到新的限制范围内
  - 效果：切换模式后立即生效，配置界面按钮显示同步更新

- **修复 TAB 边框与内容间距问题**
  - 问题：TAB 列表边框与内部元素之间没有保持间距
  - 修复：在 `PlayerTabOverlayMixin` 中添加 `TAB_CONTENT_PADDING` 常量（3px），扩展背景区域并填充背景色

### 技术改进

- **事件驱动架构**：采用事件驱动方式监听变化，避免每帧检查造成的性能问题
  - 在客户端 tick 事件中检查（每秒 20 次）
  - 使用缓存机制，只在值真正变化时调整
  - 正常情况下性能开销可忽略（简单的整数比较）

- **自动保存机制**：配置调整后立即保存到文件，确保配置文件始终保持最新状态

- **配置界面同步**：配置界面初始化时和血量模式变化时自动调整按钮显示

## 2026-04-28

### UI 改进

- **颜色选择器优化**：改进"主题样式"模块中的颜色选择器UI
  - 将 HEX 值输入框移至 HSV 面板下方，并添加"HEX"标签
  - HEX 输入框现在可以正常选中和输入，支持手动输入色值
  - 手动输入 HEX 色值后，HSV 面板光标位置会自动同步
  - 在预览框下方新增垂直透明度滑块（Alpha 条），与色相条底部对齐
  - 保留颜色预览框，位于右上角，可实时查看当前选择的颜色
  - 缩小 HSV 面板选择光标尺寸（半径从6px缩小到4px），提升精确度
  - 采用 AE 风格配色方案：背景色 #9AA0AC，边框色 #3A3A3A

### 问题修复

- **修复颜色选择器光标闪动问题**：
  - 修复在 HSV 面板底部（黑色区域）拖动时光标闪动和乱跳的问题
  - 改进 RGB 到 HSV 的转换算法，在低亮度和低饱和度时保持色相稳定
  - 修复色相条拖动到底部后继续拖动会弹到顶部的 bug
  - 限制色相值上限为 0.999，避免色相值在 0.0 和 1.0 之间的回绕跳动
  - 添加防循环更新机制，避免鼠标拖动时HSV值被RGB反向计算覆盖
  - 改进色相条边界处理，严格限制鼠标位置和色相值范围
- **修复配置界面崩溃问题**：
  - 修复自定义主题配置文件损坏时导致的 JSON 解析异常
  - 添加配置加载失败时的容错处理，自动使用默认配置
  - 修复组件初始化失败时的空指针异常
- **修复透明度条交互问题**：
  - 修复透明度条无法拖动的 bug（错误使用了X坐标而非Y坐标）
  - 改进透明度条渐变方向：从上到下为不透明→透明（更符合直觉）
  - 改进透明度条视觉效果：启用混合模式，透明部分能清晰显示棋盘格背景
  - 使用非线性渐变（平方函数），优化透明度条显示效果
  - 透明度越低的区域，棋盘格背景越明显，视觉效果更加直观

### 新增功能

- **玩家血量显示**：在 TAB 列表中玩家名称后显示血量心形图标

  - 支持半心显示

  - 切换时在动作栏显示提示信息（"TAB 列表已固定" / "TAB 列表已取消固定"）
  - 退出世界时自动清除固定状态

- 修复延迟/在线时长元素与边框右侧存在多余间距的问题
- 修复血量显示与延迟/在线时长同时开启时，心形图标与用户名间距不一致的问题

- 客户端列宽度计算缓存（FPS 提升 5-15%）
- 称号 API 缓存（API 调用减少 80%）

## 2026-04-24

### 移除功能

- **移除 `<rainbow>` 动画彩虹标签**

  如果之前使用了 rainbow 标签，请迁移到静态渐变：

  ```
  # 之前
  <rainbow>欢迎来到服务器</rainbow>

  # 现在
  <gradient #FF0000,#FF7F00,#FFFF00,#00FF00,#0000FF,#4B0082,#9400D3>欢迎来到服务器</gradient>
  ```

### 新增功能

- **渐变颜色标签 `<gradient>`**：支持 2 个或更多颜色的静态渐变
- **UI 改进**：
  - 输入框高度统一调整（标题 40px，内容 40px）
  - 修复滚动条触发范围问题
  - 优化输入体验

### 性能改进

移除 rainbow 功能后：
- 无需定期刷新 TAB 列表
- CPU 占用降低
- 网络流量减少
- 内存占用减少
