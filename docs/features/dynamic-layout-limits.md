# 动态布局限制功能

## 概述

NeoTab 现在支持根据 GUI 缩放和血量显示模式动态调整 TAB 列表的最大列数和行数。这确保了在不同屏幕尺寸和设置下，TAB 列表始终能够正确显示而不会溢出屏幕。

## 功能特性

### 混合方案

系统采用**混合方案**，结合了固定映射表和动态计算：

1. **固定映射表（安全上限）**
   - 根据 GUI 缩放级别（自动、1-6）和血量显示模式（单独/完整）提供预设的安全上限
   - 经过测试验证，确保在标准情况下稳定可靠

2. **动态计算（理论最大值）**
   - 根据实际屏幕分辨率和 GUI 缩放动态计算理论最大值
   - 自动适应特殊分辨率（如超宽屏）

3. **最终限制**
   - 取固定上限和动态计算值中的**较小值**
   - 确保既不会超出安全范围，又能充分利用可用空间

## 用户体验

### 配置界面

在配置界面的"布局分列"section中：

1. **动态按钮循环**
   - 列数按钮：从 1 循环到当前最大值，然后回到 1
   - 行数按钮：按预设值循环（5→10→15→20→25→30→35→40），但不超过当前最大值

2. **限制信息显示**
   - 实时显示当前的最大列数和行数
   - 显示推荐的列数和行数（约为最大值的 70%）
   - 格式：`当前限制：最大 X 列 × Y 行（推荐：A 列 × B 行）`

3. **智能提示**
   - 工具提示说明最大值会根据 GUI 缩放和血量显示模式动态调整
   - 帮助用户理解为什么某些值不可用

### 自动适应

- 当用户更改 GUI 缩放时，限制会自动更新
- 当用户切换血量显示模式时，列数限制会自动调整
- 如果当前配置超出新的限制，会自动调整到最大允许值

## 技术实现

### 核心类

#### `TabLayoutLimits`

混合方案的核心实现类，提供以下功能：

```java
// 获取当前最大列数（考虑血量显示模式）
int maxColumns = TabLayoutLimits.getMaxColumns(healthMode);

// 获取当前最大行数
int maxRows = TabLayoutLimits.getMaxRows();

// 获取推荐列数
int recommendedColumns = TabLayoutLimits.getRecommendedColumns(healthMode);

// 获取推荐行数
int recommendedRows = TabLayoutLimits.getRecommendedRows();

// 获取详细信息（用于调试）
String info = TabLayoutLimits.getLimitsInfo(healthMode);
```

#### `TabLayoutConfig`

布局配置类，现在使用动态限制：

```java
// 设置列数（自动应用当前限制）
config.setColumns(5);

// 设置行数（自动应用当前限制）
config.setRowsPerColumn(30);

// 获取当前最大值
int maxCols = config.getMaxColumns();
int maxRows = config.getMaxRows();
```

### 动态计算逻辑

#### 列数计算

```
可用宽度 = 屏幕宽度 - 左右边距 - 翻页箭头空间
单列宽度 = 玩家名称宽度 + 血量显示宽度 + 延迟宽度 + 时长宽度 + 内边距
最大列数 = min(可用宽度 / 单列宽度, 安全上限, 绝对上限10)
```

#### 行数计算

```
可用高度 = 屏幕高度 - 上下边距 - 标题页脚空间
最大行数 = min(可用高度 / 单行高度, 安全上限, 绝对上限50)
```

## 优势

1. **稳定性**：固定映射表确保在标准情况下不会出现显示问题
2. **灵活性**：动态计算适应特殊分辨率和配置
3. **用户友好**：自动调整，无需手动计算
4. **未来兼容**：新的 GUI 缩放选项自动支持
5. **性能保护**：绝对上限防止极端配置导致性能问题

## 调试

如果需要查看当前限制的详细信息，可以在代码中调用：

```java
String info = TabLayoutLimits.getLimitsInfo(healthMode);
// 输出示例：
// GUI Scale: 3 | Screen: 1920x1080 | Columns: 5 (safe:5, dynamic:7) | Rows: 40 (safe:40, dynamic:45)
```

这会显示：
- 当前 GUI 缩放级别
- 屏幕分辨率（GUI 缩放后）
- 最终列数限制（以及安全上限和动态计算值）
- 最终行数限制（以及安全上限和动态计算值）

## 未来改进

可能的改进方向：
