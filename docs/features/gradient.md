# 渐变颜色

NeoTab 支持 `<gradient>` 标签，可以让文字在多个颜色之间平滑过渡。

## 基本语法

```
<gradient #颜色1,#颜色2,#颜色3...>文字内容</gradient>
```

- 至少需要 2 个颜色值
- 颜色值用英文逗号分隔
- 支持任意数量的颜色值
- 每个颜色值使用 6 位十六进制格式（`#RRGGBB`）

## 使用示例

### 双色渐变

```
<gradient #FF0000,#0000FF>欢迎来到服务器</gradient>
<gradient #00FF00,#FFFF00>在线玩家</gradient>
```

### 三色渐变

```
<gradient #FF0000,#00FF00,#0000FF>RGB Gradient</gradient>
<gradient #FF0000,#FF8800,#FFFF00>火焰效果</gradient>
```

### 多色渐变

```
<gradient #FF0000,#FF7F00,#FFFF00,#00FF00,#0000FF,#4B0082,#9400D3>七色彩虹</gradient>
```

### 与样式标签组合

```
<gradient #FF0000,#0000FF><bold>重要公告</bold></gradient>
<gradient #FFD700,#FFA500><bold><underlined>金色标题</underlined></bold></gradient>
```

### 与占位符组合

```
<gradient #FFD700,#FFA500>玩家: %player_name%</gradient>
<gradient #00FF00,#FFFF00>在线: %online%/%max_players%</gradient>
```

## 技术细节

### 渐变算法

渐变使用线性插值（Linear Interpolation）算法：

1. 每个字符根据其在文本中的位置计算对应的颜色
2. 颜色在 RGB 空间中独立插值
3. 支持多段渐变（3 个或更多颜色）

### 示例计算

对于文字 "HELLO"（5个字符）和渐变 `#FF0000,#0000FF`（红到蓝）：

| 字符 | 位置 | 最终颜色 |
|------|------|----------|
| H | 0% | `#FF0000` |
| E | 25% | `#BF0040` |
| L | 50% | `#800080` |
| L | 75% | `#4000BF` |
| O | 100% | `#0000FF` |

### 性能说明

- 渐变文字会被拆分为单个字符的 Component
- 首次解析后结果会被缓存，后续调用性能提升 90%+
- 对于很长的文本（>50字符），建议适度使用渐变效果

## 常见问题

**Q: 为什么渐变标签没有生效？**

检查以下几点：
1. 确保至少有 2 个颜色值
2. 颜色值之间用英文逗号分隔
3. 每个颜色值都以 `#` 开头
4. 颜色值是 6 位十六进制数字

**Q: 可以嵌套多个渐变标签吗？**

不建议嵌套渐变标签。如果需要不同的渐变效果，请使用多个独立的渐变标签。

**Q: 渐变标签计入字符限制吗？**

不计入。渐变标签本身不计入可见长度，只有实际显示的文字计入字符限制。
