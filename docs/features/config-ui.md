# 配置界面

NeoTab 提供图形化配置界面，管理员可以通过命令打开并实时调整所有设置。

## 打开配置界面

```
/neotab config
```

> 需要管理员权限（`neotab:configure` 或 OP 2级）。

## 界面结构

配置界面分为两个标签页：

### 页面配置

包含所有 TAB 显示内容的设置：

**顶部信息**
- 标题信息开关 + 输入框（最多 32 个可见字符）
- 内容信息开关 + 输入框（最多 256 个可见字符）

**玩家列表**
- 称号功能开关
- 更好的延迟显示开关（带悬浮问号提示）
- 在线时长显示开关

**底部信息**
- 自定义信息输入框（最多 256 个可见字符）
- TPS 信息 / MSPT 信息 / 在线人数 独立开关

### 主题样式

预留标签页，后续版本将支持界面主题自定义。

## 保存配置

点击**完成**按钮保存，配置立即对所有在线玩家生效。

点击**取消**按钮放弃修改。

## 重载配置

如果直接修改了配置文件，可以使用命令重载：

```
/neotab reload
```

## 输入框说明

- 所有输入框支持富文本标签和占位符
- 标题输入框：最多 32 个可见字符（标签本身不计入）
- 内容/底部输入框：最多 256 个可见字符
- 输入框支持滚动，可输入多行内容（使用 `\n` 换行）

## 架构说明

配置界面采用分模块设计：

```
TabbedNeoTabConfigScreen（主界面）
├── PageSettingsPanel（页面设置面板）
│   ├── 顶部信息配置
│   ├── 玩家列表配置
│   └── 底部信息配置
└── ThemeStylePanel（主题样式面板，预留）
```

这种设计便于后续扩展新的配置分类。

## 扩展配置界面（开发者）

如果你是模组开发者，想要扩展配置界面，可以：

1. 在 `ConfigTab` 枚举中添加新的 Tab 类型
2. 创建新的面板类继承 `ConfigPanel`
3. 在主界面的 `init()` 方法中创建 Tab 按钮
4. 在 `getCurrentPanel()` 中添加 case 分支
5. 添加相应的翻译文本

详见[称号系统 API](../developer/title-api) 文档。
